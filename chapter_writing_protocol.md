# 逐章写作规范和执行流程

本文记录当前《量子天文学》逐章重写时采用的写作规范和执行步骤。它还不是
Codex skill，而是一份以后可以改写成 `SKILL.md` 的工作说明。

## 目标

目标是把每一章写成真正可读的科学教科书章节，而不是公式列表。完成后的章节要
让读者能沿着同一条线前进：物理图像、方程、可观测量、真实数据、适用条件和局限。

每章必须做到：

- 在公式前后解释物理图像；
- 在符号第一次重要出现处定义它；
- 不在上一节末尾提前集中定义下一节才会用到的符号；
- 给出单位和典型天体物理或仪器量级；
- 说明关键关系成立所需的假设；
- 对方程、数据值、历史判断、仪器参数和数量级例子给出文献；
- 对重要公式、标度关系和诊断量生成 Python PDF 图；
- 作为整本 LaTeX 书的一部分能够干净编译。

## 必读输入文件

写作或重写某章之前，先检查这些文件：

- `book_bible.md`: 全书范围、目标读者、术语和需要避免的判断。
- `quantum_astronomy_detailed_outline.md`: 章节设计和全书结构。
- `chapters/zh/chapter_XX.tex`: 当前章节 LaTeX 源码。
- `references.bib`: 当前使用的 ADS BibTeX 文献库。
- `citation_report.md` 和 `references.md`: 已有引文地图。
- `figures.md`, `tools/figure_catalog.py`, `metadata/figure_catalog.json`: 图目录。
- `code/chapter_XX.py` 和 `code/plot_recipes.py`: 已有绘图代码。
- `glossary.md`, `notation.md`, `continuity_log.md`: 术语、符号和连续性记录。

章节已经人工扩展后，不要再从大纲重新生成整本书，除非用户明确要求。生成器
可能覆盖已经手工重写过的章节。

## 章节写作原则

### 0. 语言声音

正文要像中文母语科学作者写的，不像翻译腔，也不像生成式摘要。理想声音是一位长期写
观测申请、技术报告和综述论文的人在讲课：句子清楚，判断有边界，例子具体，读起来有
人的节奏。

具体要求：

- 多用自然的主谓宾句，少用“构成、提供、体现、强调、揭示”这类空泛动词；
- 能写“这是因为”时，不写“这体现了……的重要性”；
- 能写“仪器测到的是事件表”时，不写“本章将会讨论事件表的重要作用”；
- 避免连续排比。并列项超过四个时，拆成两句或改成表格；
- 避免英文论文直译腔，例如“在……语境下”“扮演关键角色”“提供现实边界”；
- 不写自我解释式句子，例如“公式不是装饰”“本节将说明为什么需要解释符号”；
- 不把每段都写成同样节奏。长句解释物理链条，短句给判断或边界；
- 该下判断时直接下判断，但要把假设和数量级跟上。

可以保留技术术语，例如 event table、SPADE、Fisher 信息、readiness、null test。保留的理由是
它们是领域内搜索词或本书中的稳定概念；不要为了“中文化”把标准术语翻译到读者认不出来。

### 1. 先确定本章功能

每一章开始前，先在内部确认：

- 这一章替读者解决什么问题？
- 它给后续章节引入哪些概念？
- 它依赖前面哪些定义？
- 哪些方程是核心，哪些只是辅助？
- 哪些观测例子能让本章落到真实世界？

开头几页要让读者明白本章在书中的位置，但不要写成计划书或自我说明。避免“本章
将会说明为什么公式不是装饰”这类句子，直接进入物理解释。好的开头通常从一个
具体观测事实或一个明确矛盾开始，而不是从“本章的任务是……”开始。

### 2. 使用教科书语气

用清楚的中文写出科学教科书的节奏：

- 每段只推进一个物理想法；
- 方程出现前要有足够上下文，让读者觉得它自然出现；
- 方程之后解释每一项如何测量、单位是什么、典型量级是多少；
- 符号定义要贴着第一次真正使用它的物理情境出现，不要提前做术语清单；
- 只要可能，就使用具体观测数值；
- 避免空泛、夸张或自我解释式语言；
- 不要在每个小节后放总结段；
- 小节过碎、阅读节奏变断时要合并；
- 只有当读者确实需要跨入新物理情境时，才使用章节级过渡。

读者应该感觉是在听一位谨慎的物理学者讲课，而不是在读拼接出来的笔记。语气可以
有判断，但不要有宣传感；可以有提醒，但不要像 checklist 被塞进正文。

### 3. 把方程当作压缩后的物理图像

每个关键方程都要当作一个物理情境的紧凑模型来处理。重要方程周围的文字需要回答：

- 左边的量在数据中是什么？
- 它从光子事件表、图像、光谱、可见度点、相关直方图或探测器输出中怎样估计？
- 右边每个符号表示什么？
- 单位是什么？
- 对恒星、光学仪器、Cherenkov 阵列、光子计数器或实验室类比系统，典型值是多少？
- 这个关系需要哪些假设？
- 真实数据中最先破坏它的是什么：背景、有限时间分辨率、死时间、模型不匹配、标定漂移、稀疏基线还是选择效应？

例如写完信噪比公式后，不能停在代数关系。要继续讨论收集面积、量子效率、电子带宽、
积分时间、源星等、可见度幅度和实际极限星等。

### 4. 用物理演化串联公式

不要独立罗列公式，而要建立链条：

1. 物理设置；
2. 理想量；
3. 可观测估计量；
4. 仪器响应；
5. 极限尺度或数量级例子；
6. 系统误差；
7. 和下一个概念的关系。

以强度干涉为例：

- 从热光源的光子到达开始；
- 定义强度涨落；
- 引入 `g^{(2)}`；
- 通过 Siegert 关系或 HBT 逻辑把它连到 `|gamma^{(1)}|^2`；
- 加入有限带宽和探测器时间响应；
- 说明观测到的 excess 怎样被稀释；
- 把测得的 `|V|^2` 连到角直径或源结构。

### 5. 保持章节结构连贯

只保留物理内容真正需要的小节数量。如果几个小节在重复同一套动作，就合并。

推荐结构：

- 每章 4 到 7 个有实质内容的大节；
- 不做机械的小节总结；
- 不使用“本节小结”这种机械标题；
- 练习或计算项目可以放在章末，但必须和本章方程、数据分析相连；
- 表格适合整理单位、符号、假设和典型范围。

## 引文规范

### 1. 文献来源

天体物理、天文学、宇宙学、仪器、量子光学及相关主题的引文，以 NASA ADS 为准。

规则：

- 使用 `references.bib` 中的 ADS BibTeX 条目；
- 保留 ADS BibTeX key，通常就是 bibcode；
- ADS 能导出时，不手写 BibTeX；
- 不发明引用 key；
- 不引用没有核对过的论文；
- 永远不要把 ADS API token 写进仓库或书稿文件；
- 需要访问 ADS API 时，从环境变量 `ADS_API_TOKEN` 读取。

### 2. 不只读摘要

扩展每一章时，相关文献要尽量读到正文。优先顺序：

- 有 arXiv PDF 或 arXiv HTML 时优先读它们；
- 能访问 publisher PDF 时读 publisher PDF；
- 没有 arXiv 版本时，再使用 ADS 摘要加链接全文；
- 用综述文献把握历史背景和参数范围；
- 用仪器论文获取真实数字和观测限制。

阅读笔记要记录实际使用了什么：方程、参数值、历史判断、图的想法或局限。

### 3. 引文密度

每个大章节通常至少需要 20 到 30 篇唯一引文。如果一章同时覆盖历史、理论、仪器和
现代实验，可以更多。

引文位置：

- 来自标准结果或具体论文的重要方程都要引用；
- 数据值、望远镜参数、探测器参数和样本极限都要引用；
- 历史判断要引用；
- 现代实验或观测展示、当前限制要引用；
- 概括一个领域时引用综述；
- 首次实验、命名关系和关键数值结果引用原始论文。

避免堆引用。引文应尽量贴近它所支持的句子。

### 4. 引文检查

编辑完章节后运行：

```bash
python - <<'PY'
from pathlib import Path
import re
bib = Path("references.bib").read_text(encoding="utf-8", errors="ignore")
keys = set(re.findall(r"@\w+\s*\{\s*([^,]+)", bib))
for ch in range(1, 23):
    path = Path(f"chapters/zh/chapter_{ch:02d}.tex")
    if not path.exists():
        continue
    text = path.read_text(encoding="utf-8")
    cites = []
    for m in re.finditer(r"\\cite\w*\{([^}]+)\}", text):
        cites.extend(k.strip() for k in m.group(1).split(",") if k.strip())
    missing = sorted(set(cites) - keys)
    print(f"ch{ch:02d}: unique_cites={len(set(cites))} missing={missing}")
PY
```

完成一轮章节处理后，不应再有 `\citationneeded{}`。

## 图和 Python 代码规范

### 1. 什么时候加图

当一张图能解释某个具体方程、数据集、标度关系、观测诊断、系统效应或参数退化时，
就应该加图。

不要加装饰图。不要使用“关键方程的教学图”这类图注。图注必须说明读者能
从图中的量学到什么。

适合画成图的主题包括：

- `|V|^2` 怎样随基线和角直径变化；
- 有限时间分辨率怎样稀释 `g^{(2)}-1`；
- 源光子比例和背景怎样压低相关信号；
- 死时间和 afterpulsing 怎样扭曲短延迟直方图；
- Fisher 信息怎样随基线、光子数或系统误差地板变化；
- 相位丢失怎样造成镜像退化；
- 误差预算怎样在统计误差和标定项之间分配。

### 2. 图的风格

使用现代、干净、接近出版风格的图：

- 输出格式为 PDF；
- 字号和线宽一致；
- 配色克制；
- 坐标轴标签清楚，并带单位；
- 只有需要时才放图例；
- 不使用杂乱网格；
- 解析图不要用位图截图；
- 使用物理值，或者明确标注为教学示例值。

生成文件放在：

```text
figures/generated/chapter_XX/
```

### 3. 代码组织

每章一个 Python 脚本：

```text
code/chapter_XX.py
```

可复用绘图函数放在：

```text
code/plot_recipes.py
```

新增图时：

1. 在 `code/plot_recipes.py` 中新增或更新绘图函数；
2. 从 `code/chapter_XX.py` 调用该绘图函数；
3. 把 PDF 写入 `figures/generated/chapter_XX/`；
4. 把图插入 `chapters/zh/chapter_XX.tex`；
5. 更新 `figures.md`；
6. 更新 `tools/figure_catalog.py`；
7. 重新生成或手动同步 `metadata/figure_catalog.json`；
8. 运行本章脚本并确认 PDF 存在。

### 4. 图注标准

每个图注应该包括：

- 图里画的是什么；
- 它解释哪个物理或观测关系；
- 趋势应该怎样读；
- 有什么重要限制或注意事项。

例子：

```tex
\caption{零基线相关幅度 \(N_0\) 的标定对 \(|V|^2\) 的直接影响。左图示意校准星给出的夜间 \(N_0\) 漂移；右图显示 \(N_0\) 若偏高或偏低 8\%，所有基线上的 \(|V|^2\) 都会被整体拉伸，从而把角直径、边缘昏暗或扁率拟合推向错误值。}
```

### 5. 图文件检查

检查每个被正文引用的图文件都存在：

```bash
python - <<'PY'
from pathlib import Path
import re
for ch in range(1, 23):
    path = Path(f"chapters/zh/chapter_{ch:02d}.tex")
    if not path.exists():
        continue
    text = path.read_text(encoding="utf-8")
    paths = re.findall(r"\\includegraphics(?:\[[^\]]*\])?\{([^}]+)\}", text)
    missing = [p for p in paths if not Path(p).exists()]
    print(f"ch{ch:02d}: includegraphics={len(paths)} missing={missing}")
PY
```

## LaTeX 写作规则

沿用本书已有风格。除非用户要求，不要引入新的文档类、参考文献系统或符号体系。

规则：

- 使用 `\citep{...}` 和 ADS key；
- 数学粗体向量用 `\bm{...}`，不要用 `{\bf ...}`；
- 关键方程使用稳定标签；
- 图使用稳定标签；
- 源文件中的换行要可读；
- 过长公式优先用 `split` 或 `aligned` 环境拆开；
- 表格要紧凑，并真正解释符号、单位或假设；
- 不留下占位文字；
- 不引用不存在或已经废弃的图文件；
- 不为了总结已说过的话而新建小节。

## 逐章执行流程

### 第 1 步：确定本章局部目标

阅读章节源码和大纲中对应部分，找出：

- 当前小节结构；
- 只有公式、缺少解释的段落；
- 缺少的物理解释；
- 缺少的单位和典型范围；
- 没有文献支持的判断；
- 缺失的图；
- 缺失的系统误差讨论；
- 应该合并的重复小节。

可以为本章写一份简短工作笔记，即使不提交进仓库。

### 第 2 步：建立阅读清单

先使用本章已有引文，再通过 ADS 和 arXiv 补缺口。按用途给文献分组：

- 基础理论；
- 历史实验；
- 现代观测或实验展示；
- 仪器；
- 数据分析；
- 系统误差；
- 科学应用。

对每篇重要文献提取：

- 本章要用的主要结果；
- 相关方程或标度关系；
- 数值参数；
- 假设；
- 局限；
- 是否支持某个图的设计。

### 第 3 步：扩展正文

逐节重写正文。常用段落骨架是：

1. 物理情境；
2. 可观测量或数据产品；
3. 方程；
4. 符号和单位解释；
5. 典型数值范围；
6. 假设；
7. 系统误差或观测限制；
8. 引文。

这个骨架不应被读者看见，不要在正文里宣布“下面解释符号、单位和假设”。

### 第 4 步：合并和顺稿

初稿完成后，删除不必要的小节，让过渡自然。章节应该读起来像连续论证。如果相邻两个
小节只是符号不同，就合并。

### 第 5 步：增加图和代码

对每个核心标度关系或诊断量：

- 判断读者需要比较什么；
- 选择真实或明确标注的参数范围；
- 编写或更新 Python 绘图函数；
- 生成 PDF；
- 把图插到第一次详细讨论附近；
- 写一个真正解释图中内容的图注。

不要等整本书最后再补图。图是章节解释的一部分。

### 第 6 步：更新图目录和连续性文件

需要更新：

- `figures.md`;
- `tools/figure_catalog.py`;
- `metadata/figure_catalog.json`;
- 如果出现新概念、新依赖或潜在冲突，更新 `continuity_log.md`；
- 当术语或符号变得重要时，更新 `glossary.md` 和 `notation.md`。

### 第 7 步：编译和日志检查

编译整本书，不只编译单章，因为参考文献、浮动体和分页都会全局互相影响。

推荐编译命令：

```bash
rm -rf /tmp/qa_book_build
mkdir -p /tmp/qa_book_build
/Library/TeX/texbin/latexmk -xelatex -interaction=nonstopmode \
  -halt-on-error -file-line-error -outdir=/tmp/qa_book_build main.tex
cp /tmp/qa_book_build/main.pdf build/main.pdf
cp /tmp/qa_book_build/main.log build/main.log
cp main.bbl build/main.bbl 2>/dev/null || true
```

使用 `/tmp` 可以避免在 Dropbox 或 Overleaf 同步目录中编译时偶尔出现的 PDF 截断问题。

检查日志：

```bash
rg -n \
  -e 'LaTeX Error' \
  -e 'Undefined control sequence' \
  -e 'undefined references' \
  -e 'Package natbib Warning' \
  -e 'Citation .* undefined' \
  -e 'Reference .* undefined' \
  -e 'Missing character' \
  -e 'Overfull \\hbox' \
  -e 'Repeated entry' \
  build/main.log || true
```

目标是没有严重问题命中。

### 第 8 步：渲染 PDF 页面

把修改过的章节页渲染出来，做视觉检查：

```bash
rm -rf tmp/pdfs/chapter_check
mkdir -p tmp/pdfs/chapter_check
pdftoppm -f START_PAGE -l END_PAGE -r 150 -png build/main.pdf tmp/pdfs/chapter_check/page
magick montage tmp/pdfs/chapter_check/page-*.png \
  -thumbnail 260x368 -tile 6x -geometry +6+6 \
  tmp/pdfs/chapter_check/contact.png
```

检查：

- 没有缺图；
- 没有黑块；
- 图注可读；
- 公式没有被截断；
- 表格没有溢出页面；
- 没有明显异常空白页；
- 章节过渡和页面流正常。

### 第 9 步：最终章节清单

一章完成前确认：

- 没有 `\citationneeded{}`；
- 没有缺失 BibTeX key；
- 大章节至少有 20 到 30 篇唯一引文；
- 关键公式有物理解释、单位、量级和假设；
- 重要公式或标度关系有图；
- 图已经生成 PDF；
- 图目录已经同步；
- 本章 Python 脚本能运行；
- 本章能随整本书编译；
- 渲染页面已经目视检查；
- 没有遗留自我解释式套话；
- 没有机械的小节总结。

## 科学内容质量线

如果一段话只说下面这些，它还没有完成：

- "由公式可见";
- "这说明该量很重要";
- "该方法有广泛应用";
- "系统误差需要考虑";
- "未来可以进一步研究".

用具体内容替换这类句子：

- 哪一项在变化；
- 变化多少；
- 单位是什么；
- 对哪个仪器或观测场景成立；
- 和哪个物理尺度相比；
- 在什么假设下成立；
- 有哪个带引文的观测例子支撑。

## 好章节应该达到的状态

完成得好的章节应当具备：

- 读者能说出被建模的数据产品是什么；
- 方程和物理图像、真实仪器相连；
- 符号和单位没有被默认跳过；
- 典型尺度出现得足够多，能校准直觉；
- 引文贴近它支持的判断；
- 图解释了单靠文字会显得抽象的关系；
- 局限是具体科学限制，不是泛泛警告；
- 本章建立的词汇和符号能被后续章节复用。

## 改写成 skill 时的注意事项

将本文改写成 Codex skill 时，skill 应该包括：

- 触发场景：扩展科学教科书章节；
- 必读文件：`book_bible.md`、章节源码、大纲和引文文件；
- 强制执行 ADS 引文流程；
- 强制优先阅读全文或 arXiv 版本；
- 强制为关键方程生成 Python/PDF 图；
- 强制 LaTeX 编译和 PDF 渲染检查；
- 明确禁止保存 API token；
- 明确禁止公式列表、占位图注和逐小节总结。

skill 的流程可以严格，但产出的正文必须自然、清楚、像人写出来的科学教科书。
